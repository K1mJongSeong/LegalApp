## 1. 제품 개요 (Product Overview)

### 1.1 제품 비전
**"법률 문제, 전문가와 함께"**

LawDecode는 법률 문제로 어려움을 겪는 일반 사용자와 검증된 법률 전문가를 연결하는 **법률 상담 플랫폼**입니다. AI 기반 사건 분석과 전문가 매칭을 통해 누구나 쉽게 법률 서비스에 접근할 수 있도록 합니다.

### 1.2 제품 유형
- **플랫폼 유형**: 양면 마켓플레이스 (Two-sided Marketplace)
- **앱 유형**: 하이브리드 앱 (모바일 + 웹)
- **지원 플랫폼**: Android, iOS, Web

### 1.3 핵심 가치 제안 (Value Proposition)

| 대상 | 가치 제안 |
|------|----------|
| **일반 사용자** | 복잡한 법률 문제를 AI가 분석하고, 검증된 전문가와 쉽게 연결 |
| **법률 전문가** | 프로필 구축, 콘텐츠 관리, 상담 요청 관리를 위한 통합 플랫폼 |

## 2. 타겟 사용자 (Target Users)

### 2.1 일반 사용자 (User)
- **페르소나**: 법률 문제를 겪고 있지만 어디서부터 시작해야 할지 모르는 일반인
- **니즈**:
  - 내 상황에 맞는 법률 전문가 찾기
  - 복잡한 법률 용어와 절차 이해하기
  - 합리적인 비용으로 상담받기
  - 사건 진행 상황 추적하기

### 2.2 법률 전문가 (Expert)
- **페르소나**: 변호사, 법무사, 세무사, 노무사 등 법률 서비스 제공자
- **니즈**:
  - 온라인에서 고객 확보하기
  - 전문성과 경력을 효과적으로 어필하기
  - 상담 일정과 요청 효율적으로 관리하기
  - 콘텐츠를 통한 브랜딩

## 3. 문제 정의 (Problem Statement)

### 3.1 사용자 측 문제
1. **정보 비대칭**: 법률 서비스 품질과 가격 정보 파악 어려움
2. **접근성 부족**: 어떤 분야의 전문가가 필요한지 판단 어려움
3. **신뢰 문제**: 전문가의 자격과 경력 검증 어려움
4. **복잡한 절차**: 상담 예약부터 사건 진행까지 복잡한 프로세스

### 3.2 전문가 측 문제
1. **고객 확보**: 온라인 마케팅과 고객 유치 어려움
2. **프로필 관리**: 분산된 플랫폼에서 일관된 프로필 유지 어려움
3. **상담 관리**: 다양한 채널에서 들어오는 상담 요청 관리 복잡
4. **신뢰 구축**: 전문성을 증명하고 신뢰를 쌓기 어려움

## 4. 제품 목표 (Product Goals)

### 4.1 비즈니스 목표
- [ ] 검증된 전문가 100명 이상 확보
- [ ] 월간 활성 사용자(MAU) 1,000명 달성
- [ ] 상담 매칭 성공률 80% 이상

### 4.2 사용자 경험 목표
- [ ] 사건 등록부터 전문가 매칭까지 5분 이내 완료
- [ ] 전문가 프로필 완성률 평균 80% 이상
- [ ] 사용자 만족도 평점 4.5/5.0 이상

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 성능 (Performance)
- 페이지 로딩 시간: 3초 이내
- API 응답 시간: 1초 이내 (AI 분석 제외)
- AI 분석 응답 시간: 10초 이내

### 7.2 확장성 (Scalability)
- 동시 사용자: 1,000명 이상 지원
- 데이터 저장: Firebase Firestore 무제한 확장

### 7.3 보안 (Security)
- Firebase Authentication을 통한 안전한 인증
- Firestore Security Rules를 통한 데이터 접근 제어
- HTTPS 암호화 통신
- 개인정보 암호화 저장

### 7.4 가용성 (Availability)
- 서비스 가용성: 99.9% 이상
- Firebase 인프라 활용

### 7.5 호환성 (Compatibility)
- Android 5.0 (API 21) 이상
- iOS 12.0 이상
- 최신 웹 브라우저 (Chrome, Safari, Firefox, Edge)

## 10. 데이터 모델 (Data Models)

### 10.1 주요 Firestore 컬렉션

## 11. 프로필 완성률 계산 (Profile Completion)

### 11.1 필수 필드 (총 16개)

| 섹션 | 필드 | 개수 |
|------|------|------|
| 기본정보 | 프로필 이미지, 이름, 직업, 소속 | 4 |
| 인적사항 | 생년월일, 성별, 전화번호, 이메일, 대표전화 타입 | 5 |
| 학력사항 | 학력 1개 이상 | 1 |
| 주요분야 | 분야 1개 이상 | 1 |
| 사무실 정보 | 소속 지점, 시/도, 시/군/구, 도로명 주소 | 4 |
| **합계** | | **16** |

### 11.2 완성률 계산 공식
```
완성률(%) = (입력된 필수 필드 수 / 16) × 100
```

## 13. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| 사건 (Case) | 사용자가 등록한 법률 상담 건 |
| 전문가 (Expert) | 변호사, 법무사, 세무사, 노무사 등 법률 서비스 제공자 |
| 상담 요청 (Consultation Request) | 사용자가 전문가에게 보낸 상담 예약 요청 |
| 인증 (Certification) | 전문가의 자격 및 신원 확인 절차 |
| 프로필 완성률 | 전문가 프로필의 필수 정보 입력 정도 (%) |
| AI 분석 | GPT를 활용한 법률 사건 자동 분석 |
| 수임료 | 전문가가 사건을 맡을 때 받는 비용 |
| 성공보수 | 사건 성공 시 추가로 받는 보수 |
